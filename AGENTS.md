# AGENTS.md — SIDIKBank Architecture Guide

## Project Overview

SIDIKBank is a digital banking simulator built with TanStack Start (React SSR) on Netlify. It uses Netlify Identity for authentication and Netlify Database (Postgres) for all persistence.

## Directory Structure

```
src/
├── routes/           # TanStack Start file-based routes (pages)
│   ├── __root.tsx    # Root layout; wraps app with IdentityProvider + CallbackHandler
│   ├── index.tsx     # Login/Signup page (unauthenticated landing)
│   ├── dashboard.tsx # Main dashboard with balance + recent transactions
│   ├── transfer.tsx  # Multi-step transfer flow with fraud detection
│   ├── qris.tsx      # QRIS payment simulator with canvas-drawn QR codes
│   ├── history.tsx   # Filterable transaction history
│   └── profile.tsx   # User profile editor with security status panel
│
├── lib/
│   ├── auth.ts              # getServerUser() — wraps @netlify/identity getUser()
│   ├── identity-context.tsx # React context for client-side auth state + onAuthChange
│   ├── account.ts           # getOrCreateAccount, updateProfile server functions
│   └── transactions.ts      # doTransfer, doQrisPayment, getTransactionHistory, lookupAccount
│
├── middleware/
│   └── identity.ts   # requireAuthMiddleware for server functions
│
└── components/
    ├── BankingLayout.tsx   # Sidebar navigation layout for authenticated pages
    └── CallbackHandler.tsx # Handles OAuth/email confirmation URL hash tokens

db/
├── schema.ts  # Drizzle ORM schema — single source of truth for all tables
└── index.ts   # Drizzle client (netlify-db adapter, no connection string needed)

netlify/
├── database/migrations/    # Auto-generated SQL migrations (never edit manually)
└── functions/
    └── identity-signup.ts  # Webhook: assigns 'user' role on new signup

drizzle.config.ts  # Drizzle Kit — migrations output to netlify/database/migrations/
```

## Database Schema

| Table | Purpose |
|-------|---------|
| `bank_accounts` | One per Netlify Identity user; holds balance, account number, name, phone |
| `transactions` | All transfers and QRIS payments; tracks from/to accounts, status, fee, amount |
| `audit_logs` | Records every significant action per account |
| `fraud_alerts` | Auto-created when a transfer exceeds fraud thresholds |

## Auth Pattern

- **Server-side**: `requireAuthMiddleware` reads `nf_jwt` cookie automatically via `@netlify/identity`
- **Route protection**: `beforeLoad` in each route calls `getServerUser()` and redirects to `/` if null
- **Account provisioning**: First login auto-creates a `bank_accounts` row via `getOrCreateAccount()`
- **Client-side**: `IdentityProvider` wraps the app; `useIdentity()` exposes `user`, `ready`, `logout`

## Fraud Detection

Inline in `doTransfer()` in `src/lib/transactions.ts`:
- Transfer > 90% of balance → `high` severity → transaction status becomes `flagged` (no funds moved)
- Transfer > Rp 10 million → `medium` severity → transaction proceeds but alert logged
- Transfer > Rp 5 million → `low` severity → monitored

## Key Conventions

- All server functions use `.middleware([requireAuthMiddleware])` — never skip this
- Migrations are generated by `npx drizzle-kit generate` after schema changes, never written manually
- `formatRupiah()` is inlined per file (intentional — avoids shared utility file)
- Styling is inline styles + global CSS classes in `src/styles.css` — no component library
- TypeScript strict mode is on; watch for `noUnusedLocals` and `noUnusedParameters`

## Adding Features

1. **New page**: `src/routes/<name>.tsx` with `createFileRoute` + `beforeLoad` auth guard
2. **New DB table**: Add to `db/schema.ts`, run `npx drizzle-kit generate`
3. **New server action**: `createServerFn` with `requireAuthMiddleware` in `src/lib/`
